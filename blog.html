A bit of Git

20 August 2015

Tonight I have been looking into using GitHub through the command line. This is not supposed to be a guide but more a log of the commands needed to perform basic actions. Commands include:

  • $ git clone   - take repository from GitHub to your local computer by pasting the repo’s url.
  • $ git status   - checks whether the repo’s branch is up to date with your local branch
  • $ git branch   - tells you which branch you are currently working in
  • $ git checkout   - allows you to change branch from master to gh-pages and back
  • $ git add   - add updates and new files to be committed to GitHub
  • $ git commit -m ""   - commit the changes to your GitHub account
  • $ git push   - pushes changes onto your GitHub page

As can be seen from the URL, I am hosting this site through GitHub’s gh-pages. The basic steps in setting up your page on GitHub are:

  1. On selected repository create a new branch called gh-pages in GitHub
  2. Under Settings, choose gh-pages branch as the default branch
  3. Add files to this new branch locally, with the desired homepage set to index.html
  4. Add, commit and push these files to GitHub
  5. Check out your new website at: githubusername.github.io.repositoryname

One thing that I have not managed to get to grips with: when making a change to the master branch, the git add commit push combination works perfectly, then switch to the gh-pages branch and I am struggling to use the merge command to update the file. A bit of research tells me a post-commit hook is what I should be looking into, but for now I will stick to just using the gh-pages branch as the default and not worry about the master branch.
