A quick update

20 August 2015

Tonight I have been looking into some GitHub basics. As can be seen from the URL, I am hosting this site through GitHub's gh-pages. The basic steps have been to create a new branch called "gh-pages", then to add, commit and push files from my computer into the new branch. Now I am exploring how to commit directly from the master branch, rather than from my local directory every time. Wish me luck....

Ok, semi-success. I cannot yet commit directly from the master branch, but I have managed to eradicate the issues in my gh-pages index.html file. The commands at the bottom of this post proved most useful

I can now clone a repository from github onto my computer, update it on my computer using Sublime Text 2, then upload the new file(s) to GitHub all via the command line.

Step one: navigate to desired location for repository to be cloned into your computer using the commands ls and cd. Use mkdir new-folder-name to create a new folder. Next we want to clone from git:

$ git clone "copy the url of the repository here, excluding quotation marks"

Now you have the full repository uploaded onto your computer, you can "open" adapt and then save files. Once saved, you want to update your GitHub repository:

$ git add -A
$ git commit -m "some note for the update eg. New blog post"
$ git push origin master


Some other useful git commands:
git status, git branch, git checkout "branchname", git rm "filename", git push
