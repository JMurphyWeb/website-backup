Regular Expressions For Beginners

21 September 2015

Beginners Guide To Regular Expressions: Eloquent JavaScript chapter 9

Regular Expressions are almost impossible to decipher from studying other peoples code, I wanted to write a quick guide to how you can go about learning them in just a few days.

The Basics: Matching and Skipping

First of all you want to get comfortable with using regular expressions to match and skip certain strings. Regexone is the place to start, a brilliant resource not to be missed. Do all of the 'lessons' and 'exercises', if you get stuck don't forget you can check the solution at the bottom of each page.

Implementing: Methods

Now that you have a good grasp of how to precisely match specific strings, it is time to take a look at how you can use these new skills. Each time you create a regexp object, some useful methods are created with it. Now you can start to read Eloquent JavaScript - Chapter 9, which combined with the following exercises on Codewars should let you up nicely to exploit the potentially very efficient, but not always elegant nature of regular expressions.

Regex101 is an additional extremely useful resource for completing the exercises for live testing of your regular expressions.

Exercises:
  • Page 165. Testing for matches: valid PIN code
  • Page 167. Repeating parts of a pattern: Valid number to 2 decimal places
  • Page 168. Matches and groups: Where's Wally?
  • Page 176. Replace method: Unscramble Eggs
  • Page 176. Replace method: Heggeleggleggo
  • Page 176. Replace method: HTMLify my text
  • Page 177. Greed: Dodgy delete
  • Page 179. Dynamically creating RegExp objects: Error catcher
  • Page 181. Looping over matches: Find the names

You should be more than ready to complete the Eloquent JS exercises at the end of chapter 9.

A bit of Git

20 August 2015

Tonight I have been looking into using GitHub through the command line. This is not supposed to be a guide but more a log of the commands needed to perform basic actions. Commands include:

  • $ git clone   - take repository from GitHub to your local computer by pasting the repo’s url.
  • $ git status   - checks whether the repo’s branch is up to date with your local branch
  • $ git branch   - tells you which branch you are currently working in
  • $ git checkout   - allows you to change branch from master to gh-pages and back
  • $ git add   - add updates and new files to be committed to GitHub
  • $ git commit -m ""   - commit the changes to your GitHub account
  • $ git push   - pushes changes onto your GitHub page

As can be seen from the URL, I am hosting this site through GitHub’s gh-pages. The basic steps in setting up your page on GitHub are:

  1. On selected repository create a new branch called gh-pages in GitHub
  2. Under Settings, choose gh-pages branch as the default branch
  3. Add files to this new branch locally, with the desired homepage set to index.html
  4. Add, commit and push these files to GitHub
  5. Check out your new website at: githubusername.github.io.repositoryname

One thing that I have not managed to get to grips with: when making a change to the master branch, the git add commit push combination works perfectly, then switch to the gh-pages branch and I am struggling to use the merge command to update the file. A bit of research tells me a post-commit hook is what I should be looking into, but for now I will stick to just using the gh-pages branch as the default and not worry about the master branch.
